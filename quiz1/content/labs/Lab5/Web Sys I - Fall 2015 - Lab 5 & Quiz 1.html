Quiz 1 - jQuery, JSON and AJAX - Lab 5

Description

    • Using what we have learned you are to create a new site for this course.
    • The site should be well structured and should allow for content to be dynamic as the course progresses.
    • It will be placed in the folder named websysquiz1

    • Part 1

    • In the websysquiz1 folder you will need to create a homepage named index.html
    • This page will be your websys site's 'landing page' and will ultimately contain a menu of links to relevent material for the class.
    • Make sure you allow for a structure similar to the following;
      • root
      • -resources
      • -content
      • --lectures…only a representitive sample…
      • --labs
      • ---labx
      • ----labxsolution
      • --homeworks
      • ---homework1
      • ----homework1solution
      • ---homework2
      • ----homework2solution
      • --quizzes
      • ---quiz1
      • ----quiz1solution
      • …and so on…
    • Content: Take the HTML files from each lab, homework assignment and this quiz from LMS, and place it the appropriate directory.
    • Make sure your css, js and resource files are all in the appropriate directories/folders.
    • Modify the htmls from LMS to follow a consistent style for all assignments using external CSS apply style. (you may use this file as an example)

    • Part 2 : jQuery & AJAX & JSON

    • You are to create a JSON file with data necessary to reference links to data to be represented on your site.
    • The file should contain at least, the file name, and a description of the content as well as a type (ie. assignment, lecture, example, etc). You may wish to add more, but do not embed HTML tags in your JSON file.
    • You will read this data from the JSON file, using jQuery and AJAX to populate your websys site menu with links to the content pages for the class materials. Your class materials should allow for any file type to be displayed. html files will open in the browser - you need to handle the HTML/CSS styling of the content, other content (ie text files or powerpoint presentations) may open in the browser or trigger a download - your choice.
    • All html will need to be styled according to your site. The HTML from LMS is inconsistent - this will never do for a website.
    • Suggestion: Remove all formatting tags from the individual assignment files and then create a template for your site with a simple css, and some HTML which may be included. Then you can read in the plain text of the assignment and format it in your page.
    • make sure you come up with a standard template files for the site. If you want to have a couple (ie labs.css, lectures.css, header.html, footer.html, etc) that is good, but be careful as you want to have standardization across the site
    • In the future, one should be able to add the content to the appropriate directory(ies), enter an item in the JSON file, and the site should be updated accordingly.

    • Part 3 : jQuery UI

    • By now you should have a website that reads from a JSON file to build a main page with menu items accessing different content pages for our site. Some of those pages also have links to other pieces of content - zip files, html files, etc...)
    • You are to peruse the jQuery UI menu and style options and choose something you like. Then you are to incorporate that/those function(s) into your website.
    • What we want to do now, is create a menu for our main page, which will allow us to get to the content. A potential example might be that when a user lands on the page, they see some information about the course and/or the site. Then they see a menu which lets them select 'LABS'. after selecting labs, and are brought to a page with all of the labs listed. When they hover over a lab name, a menu, pop-up, etc opens to show them a description of the lab along with a list of hyperlinks which can be selected to access the files for download.
    • Be creative! Here are a couple of examples...
    • tabs
    • selectable menu
    • There are lot more - don't just use these....

    • Summary

    • You are going to build a dynamic website for Websys
    • When the page is opened it will display a page that describes the course, and provides access to the content from the semester
    • The page will link to the lectures, labs, homeworks, quiz and their solutions from the semester.
    • The items/links on the page will come from a JSON file which you will create.
    • The assignment instructions should display on the page in a consistent and structured manner.
    • The lectures and other non html content may be pulled as is for download - For Example: If lecture 1 is a PowerPoint file, then it should be located in the appropriate directory, there needs to be an entry in the json file, and when the page loads, there should be a link to the file (use target="_blank"). Beyond that, no functionality is required. (if your browser is setup correctly the file should download)
    • There must be a README.md file included which explains the logic for your site, and your observations from this lab. Additionally, you should explain your choice for your jQuery choice and JSON format. Remember, creativity is important!
